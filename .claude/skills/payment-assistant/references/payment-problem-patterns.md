# 游戏外充值问题类型匹配模式说明

本文档描述各种游戏外充值问题类型匹配的方式和特殊注意事项。

## 问题类型匹配模式

### 查询订单是否发货 (query_order_status)

**问题内容 常见模版**:
```
{order_id}这笔订单是否发货呢？ 
查询{order_id}这笔订单是否到账呢？
```

**问题特点**:
- 通常包含是否发货，是否到账字眼

**示例**:
- `791000803PP022260501164943000001这笔订单是否到账呢`
---

### 查下订单是否充值到xx游戏的，再给下账号 (query_order_game_and_account)

**问题内容 常见模版**:
```
{order_id}查下这笔订单是否充值到{game_name}的，再给下账号
```
**问题特点**:
- 通常包含是否充值到，给下账号字眼

**示例**:
- `791000803PP022260501164943000001这笔订单是否到账呢`
---


## 常见问题

### Q: 为什么匹配问题类型失败？

可能原因：
1. **问题描述不清晰** - 给出不清晰的点，让用户给出信息
2. **用户给的图片信息** - 图片信息不满足问题处理要求

### Q: 匹配问题类型失败，如何处理？

1. 提示用户需要给出那些信息，例如：问题描述缺少订单号、请提供订单截图等。
